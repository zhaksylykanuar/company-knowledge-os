# Runbook: Manual MVP Pilot

## Status

This runbook is historical/supporting. The canonical MVP path is now the
GitHub-first flow in the root master playbook and execution plan.

## First Check: Synthetic dry run

Synthetic dry run paths may still be useful for local safety checks. They are
not the current product gate and must not be used to expand Telegram/digest
behavior before the GitHub-first MVP is usable.

## Current daily founder digest v2 loop

The current daily founder digest v2 loop is preserved as post-MVP traceability,
not as the active execution spine. It is not a production scheduler and does not
approve outbound delivery.

## Human Approval Boundary

All write, send, and delivery behavior remains human-approved and separately
gated. If evidence is missing, report insufficient evidence rather than
inventing facts.
